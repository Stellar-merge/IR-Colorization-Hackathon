# Global Guidelines

## Language and Clarity
- **Target Audience:** Undergraduate students, hackathon mentors, and judges.
- **Simplicity:** Use simple, clear, and beginner-friendly English throughout the entire project. Avoid overly technical or academic wording unless necessary. Keep explanations short and clear. Prefer readability over sophisticated vocabulary. Do not use complex AI jargon when a simpler term can be used.
- **Code Comments:** Every important function should include comments explaining: What the function does, Input parameters, Output, and Important logic (if any). Write comments as if you are explaining the code to a teammate who is new to the project.
- **Function Documentation:** Use simple docstrings.
- **Variable Names:** Use meaningful, descriptive names (e.g. `input_image`, `inference_time` rather than `img1`, `temp`).
- **Clean Code:** Keep code modular, well-organized, and follow clean coding practices.

## Messages and Text
- **Console Messages:** Use clear, straightforward messages (e.g. `Loading model...`, `Running inference on test images...`). Avoid vague or overly technical messages.
- **Error Messages:** Make them understandable and actionable (e.g. `Model file not found. Please check the path to the trained weights.` instead of `Checkpoint deserialization failed.`).
- **UI Text:** All labels, buttons, notifications, and messages should use simple English.

## Documentation (README)
- Write the README in beginner-friendly English.
- Include the following sections: Project overview, Problem statement, Features, Folder structure, Installation, How to run, Training (if applicable), Testing, Evaluation, Results, and Future work.
- Explain concepts in a way that a third-year engineering student can understand. Whenever a complex algorithm is used, briefly explain why it is used before explaining how it works.

**Overall Goal:** The final project should look professional while remaining easy to read and understand. Every file should feel like it was written by a skilled engineer who values clarity, simplicity, and maintainability over unnecessarily complex language.
